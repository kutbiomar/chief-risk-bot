# Frontend Scaffold

This frontend preserves the lightweight static-MVP shape of the existing product while switching the product narrative to private-markets operations.

## Implemented Pages

- `index.html`
- `login.html`
- `onboarding.html`
- `portfolio.html`
- `cockpit.html`
- `holdings.html`
- `liquidity.html`
- `documents.html`
- `briefings.html`

## Shared Assets

- `_app.js`: session handling, API client, and page-level behavior
- `_mvp.css`: shared layout, cards, forms, tables, and briefing UI styles

## Still Missing

- richer table-editor interactions
- deeper reconciliation diff controls
- upload queue polling and background-job UX
- portfolio editing flows from the frontend beyond onboarding
